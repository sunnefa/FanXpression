Requirements

All you need to install FanXpression is Apache, PHP and MySQL. If you're not sure whether PHP and MySQL are installed on your server, ask you hosting company.

You also need to be a big fan of something, a celebrity, movie, band, book, sports team or anything else you can think of.

Installing FanXpression

After you've downloaded FanXpression, unzip the archive and upload everything to the public_html (in some cases htdocs or www) folder on your server. Make sure all the files were uploaded correctly.

To install FanXpression navigate to http://www.whateveryourdomain.is/install.php and follow the instructions.

You will need to know your database username, database password, database name and host name. You can get these through your cPanel (if you have one) or by asking your hosting company.

If the install failed for some reason, delete all the folders in your fanx_uploads folder (if there are any), remove all the database tables that may have been created from your database (if you don't know how to do this you can ask your host to do it for you) and attempt the install again.

Themes

To make your own theme, you can either start from scratch or duplicate the default theme and edit it.

In addition to the functions listed below, you can of course use any PHP functions you wish in your theme.

Theme files

The following files are mandatory for a theme to work. If the file is not found in the selected theme, a file from the default theme will be used instead.

  • header.php

    This file is where you put all your header code, the start of your html and body tags, your header, title and meta tags and include any css style sheets or javascript files you are going to use. This is also file in which the main navigation is, and a good place to put a top banner.

    The following functions are available in this file:

    • main_menu()

      This function prints the main navigation menu, with a link to the home page (if you wish) and any supercategories you have created as well as active sections if they have not been sorted into a supercategory. This function takes an optional argument, where you can specify which tags to put before the whole menu, which tags to put after the whole menu and which tags to put before and after each link.

      Example: 

      <?php main_menu(array(
    'before_menu' => '<ul class="menu">',
    'after_menu' => '</ul>',
    'before_link' => '<li>',
    'after_link' => '</li>',
    'include_home_link' => true
)); ?>

    The following variables are available in this file:

    • $site_title - This variable holds the title of the site as per your settings as well as the title of the current page, post, category or section being viewed.
    • $theme_path - This variable holds the full URL to your theme, making it useful when creating links or for use in the src attribute on img tags.

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $site_title; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

  • footer.php

    This file is where you put the end of your html code. The closing html tag, the closing body tag and any footer message you might want to show, such as a copyright notice or a link to the top of the page. All the same functions and variables are available to this file as the header.php file above.

  • post.php

    This file is responsible for showing a single post, both when viewing single posts and when viewing multiple posts. In this file is where you put the post title, the post content, tags, author, etc.

    The following variables are available to this file:

    • $permalink - The link to view the single post with comments
    • $title - The title of the post
    • $content - The posts's content
    • $date - The date the post was made
    • $username - The name of the user who made the post
    • $tags - A comma separated list of the tags this post has

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $title; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

  • 404.php

    This file is where you put whatever message you would like displayed if a file or post was not found. All the same functions and variables are available to this files as the header.php file above.

  • comments.php

    This file is responsible for showing a single comment. In this file is where you put the commenter name, their website, email address and the comment itself.

    The following variables are available to this file:

    • $author_name - The name of the comment's author
    • $author_website - The url of the comment author's website
    • $author_email - The email address of the comment's author
    • $comment - The comment's content

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $author_name; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

  • comments_form.php

    This file is responsible for showing the form through which users can add new comments to posts. This file requires some specific html attributes to be present in the form elements, so it is not recommended to edit it unless you are absolutely sure you know what you are doing.

    If you do edit it, please be careful not to change any of the form, textarea or input tags.

  • page.php

    This file is responsible for showing a single page. In this file is where you put the page title, the page content, the page's author, the date the page was created etc.

    The following variables are available to this file:

    • $title - The title of the page
    • $content - The page content
    • $user - The name of the page's author
    • $date - The date the page was created
    • $category - The name of the category the page belongs to

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $title; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

  • categories.php

    This file is responsible for showing a single category. In this file is where you put the category name and description and a list of it's child categories, pages and sections.

    The following functions are available to this file:

    • show_category_links()

      This function prints out a list of the category's child categories, pages and sections. This function takes one required argument, $links, and one optional argument where you can specify what HTML tags to put before and after the whole list and before and after each link.

      Example:

      <?php show_category_links($links, array(
    'before_link_list' => '<ul>',
    'after_link_list' => '</ul>',
    'before_link' => '<li>',
    'after_link' => '</li>'
)) ?>

    The following variables are available to this file:

    • $name - The name of the category
    • $description - The description of the category

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $name; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

  • tag.php

    This file is responsible for showing a single tag. In this file is where you put the tag name and description.

    The following variables are available to this file:

    • $name - The name of the tag
    • $description - The description of the tag

    To be able to use these variables you will need to use PHP's echo function. Example:

    <?php echo $name; ?>

    You can use echo inside any html structure you choose. If you are unsure, you can use the default theme as a reference.

Frequently Asked Questions

  1. Q: What is FanXpression?

    A: FanXpression is a content management system intended to be used to create fansites (though if you can think of other uses, by all means go for it). It has several features that allow fansite webmasters to more easily manage their fansite, such as themes, posts, pages and different sections, such as discography.

  2. Q: Who can use FanXpression?

    A: Anyone who would like to create a fansite can use FanXpression. You only have to make sure your server meets all the Requirements before you start using it.

  3. Q: What are extensions?

    A: Extensions are a way for you to extend the functionality of FanXpression by adding little extra bits. As of version 1.0 Beta, the adding of new extensions is not possible, but FanXpression comes with one extension installed, discography.

    In the future, the plan is to make more of those, allowing webmasters to only install the extensions they need (eg. you wouldn't need functionality for discography if your fansite is about a book).
